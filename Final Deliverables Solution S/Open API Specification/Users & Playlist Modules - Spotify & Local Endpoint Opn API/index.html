Auths

Up 
post /register
Register a new user (addUser)
Creates a new user account in the VIRE App. This endpoint is used for user registration.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body User (required)
Body Parameter — Payload containing new user details.

Return type

User

Example data

Content-Type: application/json
{
  "password" : "tUx!Passw0rd",
  "last_name" : "Doe",
  "creation_date" : "13-08-2000",
  "first_name" : "John",
  "email" : "tuxi@gmail.com"
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

201

User successfully registered and added to the system. User

400

Invalid input, such as missing or invalid user details. StatusRequest
Up 
post /login
User login (loginUser)
Authenticates a user in the VIRE App. It validates user credentials and provides access.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body UserLog (required)
Body Parameter — User login credentials.

Return type

StatusRequest

Example data

Content-Type: application/json
{
  "messageRequest" : "operation performed",
  "statusCode" : "200"
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

User successfully logged in. StatusRequest

400

Authentication failed due to invalid credentials or input. StatusRequest

Users

Up 
get /main/user/{userId}
Get user information (getUserById)
Retrieves detailed information about a user based on their unique userId.

Path parameters

userId (required)
Path Parameter — Unique identifier of the user. format: uuid

Return type

User

Example data

Content-Type: application/json
{
  "password" : "tUx!Passw0rd",
  "last_name" : "Doe",
  "creation_date" : "13-08-2000",
  "first_name" : "John",
  "email" : "tuxi@gmail.com"
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Successfully retrieved user information. User

404

User not found for the provided userId. StatusRequest
Up 
get /users/{userId}/playlists
Get user playlists (getUserPlaylists)
Retrieves all playlists associated with a specific user, identified by their userId.

Path parameters

userId (required)
Path Parameter — Unique identifier of the user whose playlists are being retrieved. format: uuid

Return type

array[Playlist]

Example data

Content-Type: application/json
[ {
  "member_id" : "046b6c7f-0b8a-43b9-b35d-6489e6daee91",
  "name" : "tuxiPlaylist",
  "id" : "046b6c7f-0b8a-43b9-b35d-6489e6daee91",
  "creation_date" : "13-08-2000"
}, {
  "member_id" : "046b6c7f-0b8a-43b9-b35d-6489e6daee91",
  "name" : "tuxiPlaylist",
  "id" : "046b6c7f-0b8a-43b9-b35d-6489e6daee91",
  "creation_date" : "13-08-2000"
} ]

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Successfully retrieved playlists for the user.

404

User not found for the provided userId, or no playlists available. StatusRequest
